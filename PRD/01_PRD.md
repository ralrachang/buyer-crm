## 1. 제품 개요

### 한 줄 요약
꼬마빌딩/상업용 부동산 중개 담당자가 매수 고객 정보와 상담 이력을 웹에서 안전하게 관리하는 개인 CRM 도구.

### 해결하는 문제
- 고객 정보가 메모장·엑셀·카카오톡에 분산되어 있어 상담 전 맥락 파악에 시간이 걸림
- 팀원과 정보 공유 시 파일 전달 or 구두 인계로 오류 발생
- 고객별 상담 히스토리가 쌓이지 않아 "이 고객이 언제 뭘 원했는지" 기억에 의존

### 핵심 가치
- **@멘션 타임라인**: 고객별 상담 내용을 시간 순서로 기록 — 언제든 맥락을 즉시 복원
- **패스워드 하나로 팀 공유**: 복잡한 계정 없이 비밀번호만 알면 접속 가능
- **꼬마빌딩 특화 필드**: 매입 목적·법인 보유·매수의지 등 상업용 부동산 딜에 필요한 정보 구조화

---

## 2. 사용자

### 주요 사용자
- **누구**: 꼬마빌딩/상업용 부동산 중개 담당자 (1인 또는 소규모 팀, 최대 50명 고객 관리)
- **상황**: 고객 전화/미팅 전후, 새 매물 발굴 시 "이 고객에게 맞나?" 확인할 때
- **목표**: 고객 정보를 한 곳에서 빠르게 조회하고, 상담 내용을 타임라인으로 축적

### 사용자 시나리오
1. 매수 상담 전 담당자가 웹페이지에 접속해 패스워드 입력
2. 고객 목록에서 @고객이름을 클릭해 프로필·타임라인 확인
3. 상담 후 타임라인에 @멘션(위치/매물명)과 내용 입력 → 피드에 자동 추가

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 패스워드 로그인 | 공유 비밀번호 1개로 전체 접근 제어 | P1 (MVP) | 간단 |
| 고객 프로필 관리 | @이름 기반 카드, 9개 기본정보 CRUD | P1 (MVP) | 간단 |

| 매수의지 레벨 시각화 | 1-5 레벨 색상/배지로 목록에서 한눈에 확인 | P2 | 보통 |
| 메모 검색 | 이름/지역/내용 기반 타임라인 검색 | P2 | 보통 |
| 매물 매칭 메모 | 고객에게 제안한 매물 정보 기록 | P3 | 복잡 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
패스워드 입력 → 고객 목록 → 고객 상세 → 타임라인 작성 → 저장
```

### 상세 흐름
1. **로그인**: 패스워드 입력 → 세션 유지 (브라우저 닫으면 재인증)
2. **고객 목록**: @이름 + 매수의지 레벨 + 선호지역 한눈에 보기, 고객 추가 버튼
3. **고객 등록/편집**: 9개 기본정보 입력 폼 (이름, 매입목적, 매수의지, 보유현금, 선호지역, 고객컨셉, 직업, 나이, 법인보유)
4. **고객 상세**: 상단 프로필 카드 + 하단 타임라인 피드
5. **타임라인 입력**: @멘션 필드 + 내용 텍스트박스 → 엔터/버튼으로 등록

## 5. 성공 기준

- [ ] 패스워드 없이는 어떤 데이터도 볼 수 없음
- [ ] 고객 등록 후 3초 이내에 타임라인에 글 남길 수 있음
- [ ] 모바일 브라우저에서도 타임라인 입력 가능
- [ ] 팀원이 같은 URL로 접속해 동일한 데이터 확인 가능

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.

- 개인별 계정 관리 — 이유: 공유 패스워드로 충분, 복잡도 증가
- 매물 정보 DB 연동 — 이유: Phase 3에서 메모 형태로 구현
- 이메일/문자 발송 기능 — 이유: 범위 초과
- 알림/리마인더 기능 — 이유: Phase 3 고도화 대상
- 모바일 앱 (iOS/Android) — 이유: 웹 반응형으로 충분
